# Exposure-mode delta — `intermediate` vs `intermediate_instructor`

Both bundles were generated from the same recipe / seed / difficulty.  The
only differences are produced by the **exposure layer** (`leadforge.exposure`):
filtering, redaction, and metadata emission.

This file is a pre-computed, deterministic diff so reviewers don't have to
run `jq` themselves.  Generated by the same `build.sh` that produced the
bundles.

## `manifest.json` — keys with different values

| key | `student_public` | `research_instructor` |
|---|---|---|
| `exposure_mode` | `"student_public"` | `"research_instructor"` |
| `redacted_columns` | `["current_stage", "is_sql"]` | `[]` |

All other manifest fields are identical (modulo `generation_timestamp`
which is wall-clock, and per-file `sha256` digests which differ because
the redacted columns produce different table bytes).

## Filesystem delta

**Files only in `intermediate/` (student_public):**

```
lead_scoring.csv             # flat train+valid+test convenience export
```

**Files only in `intermediate_instructor/` (research_instructor):**

```
metadata/graph.graphml       # hidden DAG (lead → motif → outcome wiring)
metadata/graph.json          # JSON mirror of the graph
metadata/latent_registry.json   # latent scores + provenance per lead
metadata/mechanism_summary.json # per-node mechanism parameters
metadata/world_spec.json     # full world specification
```

Everything under `metadata/` is **hidden truth** that students should not
see.  `student_public` mode strips the entire directory at bundle write
time (see `leadforge.exposure.metadata.write_metadata_dir`).

## Column-level delta — what `redacted_columns` actually removes

Both columns were stripped from `student_public` because measurement
showed they were near-deterministic for the label and would let a
single-feature classifier solve the task:

- **`current_stage`** — contained terminal funnel stages (`closed_won` /
  `closed_lost`) for converted leads at the snapshot anchor.  Direct
  label leakage.
- **`is_sql`** — measured P(converted | is_sql=False) ≈ 0.011 at
  advanced tier; effectively a one-rule classifier for the negative
  class.  See `leadforge/CHANGELOG.md` "Bundle schema v3" for the
  full per-tier numbers.

Redaction is applied uniformly to every published parquet under both
`tables/` and `tasks/` (so a student joining the raw `tables/leads.parquet`
back into their feature set cannot reconstruct the columns either).
The instructor bundle carries both columns in full, in every table.
